# SIGame-RS

Аналог **СВОЯ ИГРА** (SIGame / Jeopardy) на Rust для Linux.

Игра состоит из двух частей:

- **`sigame-server`** — headless-сервер (без графики), запускается в терминале и
  хранит всё состояние партии. На нём держится игра, к нему подключаются все.
- **`sigame-client`** — графический клиент (Tauri). В нём играют и **ведут**
  игру, а также **редактируют паки** вопросов.

## Возможности

- 🎯 Классический игровой процесс: табло (темы × стоимости), гонка кнопок
  (кто первый нажал — тот отвечает), начисление/штраф очков.
- 🏁 Финальный раунд: вычёркивание тем → тайные ставки → тайные ответы →
  вскрытие ведущим.
- 🖼️ Вопросы с **картинками, видео и звуком** (медиа раздаётся сервером по сети).
- ✏️ Встроенный **редактор паков**: темы, вопросы, медиа, ответы.
- 🌐 Сетевая игра: ведущий и игроки подключаются к серверу по `адрес:порт`.
- 📦 Свой формат паков **`.sgpack`** (zip + `pack.json` + `media/`).

## Как сыграть (кратко)

1. Кто-то один запускает сервер с паком:
   ```bash
   sigame-server demo/demo.sgpack
   ```
   Сервер слушает порт **7777** (игра) и **7778** (медиа).
2. Все открывают графический клиент (`sigame-client`), вкладка **«Игра»**.
3. **Ведущий** подключается, поставив галочку «Я ведущий».
   **Игроки** подключаются под своими именами без этой галочки.
4. Ведущий жмёт «Начать». Играем!

Подробности — в [docs/](docs/) и в разделе ниже.

## Установка

### Готовые сборки (проще всего)

Скачайте со страницы [**Releases**](../../releases):

- `sigame-server` — один исполняемый файл, сделайте его исполняемым и запускайте:
  ```bash
  chmod +x sigame-server
  ./sigame-server путь/к/паку.sgpack
  ```
- `SIGame-RS_*.deb` — графический клиент для Debian/Ubuntu:
  ```bash
  sudo apt install ./SIGame-RS_*.deb
  ```
  либо `SIGame-RS_*.AppImage` — один файл, `chmod +x` и запуск.

### Сборка из исходников

Нужны: [Rust](https://rustup.rs/) и системные библиотеки Tauri (webkit2gtk и пр.).

```bash
# сервер (быстро, без системных зависимостей)
cargo build --release -p sigame-server
# бинарь: target/release/sigame-server

# клиент (нужны зависимости Tauri, см. ниже)
cargo tauri build
# пакеты: target/release/bundle/
```

Удобный скрипт, собирающий всё сразу: [`scripts/build-linux.sh`](scripts/build-linux.sh).

Системные зависимости клиента (Ubuntu/Debian):

```bash
sudo apt install libwebkit2gtk-4.1-dev build-essential curl wget file \
  libxdo-dev libssl-dev libayatana-appindicator3-dev librsvg2-dev
```

Разработка клиента с горячей перезагрузкой: `cargo tauri dev`.

## Игра с друзьями по сети

Сервер слушает `0.0.0.0`, то есть доступен по локальной сети сразу. Игроки в той
же сети подключаются на **локальный IP** хоста сервера (узнать: `ip addr`),
порт `7777`.

Через интернет нужно либо пробросить порты `7777` и `7778` на роутере, либо
использовать туннель/VPN (например, `tailscale` или временный туннель). P2P без
ручной настройки запланирован на будущее (см. [TODO.md](TODO.md)).

## Структура проекта

```
core/    — чистая логика: модель паков и движок правил (без сети и графики)
server/  — headless-бинарь sigame-server (TCP + раздача медиа по HTTP)
client/  — клиент Tauri (графика, игра, редактор паков); фронтенд на HTML/CSS/JS
demo/    — демонстрационный пак
docs/    — проектная документация (архитектура, протокол, правила, формат паков)
```
