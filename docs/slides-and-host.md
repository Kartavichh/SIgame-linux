# Слайды и управление ведущим (Этап 10)

Документ фиксирует решения по приближению к SIGame: сценарий вопроса из
слайдов, ручное управление ведущего, меню настроек, бегущая полоса времени.

## Модель контента: слайды — РЕШЕНО

Старая жёсткая схема вопроса (`content: Vec<Content>` + `answer: String`)
заменяется на сценарий из слайдов:

```
Slide    { items: Vec<Content> }      // всё на слайде показывается одновременно
Question {
    price, kind,
    question_slides: Vec<Slide>,      // показываются ДО момента ответа
    answer_slides:   Vec<Slide>,      // показываются ПОСЛЕ (разбор/ответ)
}
```

- `Content` (текст/изображение/аудио/видео) переиспользуется как «блок» слайда.
- «Момент ответа игроков» = граница между `question_slides` и `answer_slides`.
- Число слайдов не ограничено (в разумных рамках).

### Совместимость со старыми паками
Загрузка терпима к старому формату: если у вопроса есть поля `content`/`answer`,
они конвертируются — `content` → один слайд в `question_slides`, текстовый
`answer` → один текстовый слайд в `answer_slides`. Новые паки пишутся уже в
слайдовой схеме.

## Управление ведущим — РЕШЕНО

Поток вопроса (ручной, как в SIGame host-режиме):
1. Игрок выбирает клетку → открывается вопрос, показан 1-й слайд `question_slides`.
   **Ответ не начинается сразу.**
2. Ведущий листает слайды: «Далее» / «Назад».
3. Кнопки для ответа открываются по одному из режимов (настройка, см. ниже).
4. Гонка кнопок → ответ → ведущий жмёт «верно/неверно».
5. Показываются `answer_slides` (ведущий листает) → возврат к выбору.

Панель ведущего: Далее/Назад по слайдам, «Открыть кнопки», верно/неверно,
назначить отвечающего, ручная правка счёта, пропустить вопрос.

## Настройки партии (отдельное меню) — РЕШЕНО

Настройки выносятся из лобби в **отдельное меню «Настройки»** (доступно ведущему).
Состав (`GameSettings`):

- **Режим открытия кнопок** (`buzz_mode`): `manual` (ведущий жмёт «Открыть
  кнопки») **или** `auto_after_last` (открываются автоматически после последнего
  слайда вопроса). Выбор в настройках.
- **Фальстарт** (`false_start`): включён/выключен. Нажатие до открытия кнопок
  блокирует игрока на **настраиваемую длительность** (`false_start_block_secs`).
- **Время на нажатие** (`buzz_time_secs`) и **время на ответ**
  (`answer_time_secs`) — выносятся в меню настроек (раньше жили только в
  `GameConfig`).
- **Кот в мешке** (`cat_must_give`) и **удвоение «без риска»** (`no_risk_double`) —
  как на Этапе 9.

Автоигра (автолистание раундов) — **решено НЕ делать** (редко используется).

## Бегущая полоса времени — РЕШЕНО

Вокруг слайда — бегущая белая полоса (как в SIGame), визуализирующая отсчёт
времени на нажатие/ответ. Длительности берутся из настроек (`buzz_time_secs` /
`answer_time_secs`). Отсчёт ведёт сервер (как и прочие таймеры), полосу рисует
клиент по данным снимка.

## Редактор — РЕШЕНО: полноценный визуальный редактор слайдов

Редактор паков получает работу со слайдами: добавить/удалить/переупорядочить
слайды в `question_slides` и `answer_slides`, на каждом слайде — несколько
блоков (текст/медиа) в произвольном порядке.
