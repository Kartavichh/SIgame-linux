# Как сыграть вместе в одной локальной сети (Wi-Fi/LAN)

> Сначала установить приложение — см. [install.md](install.md).

Короткая инструкция: один человек — **ведущий** (поднимает игру на своём
компьютере), остальные — **игроки** (подключаются к нему). Все должны быть в
**одной сети** (один Wi-Fi или один роутер).

---

## Часть 1. Ведущий (делает один человек)

### 1. Запустить игру
1. Открыть приложение → вкладка **«Игра»**.
2. Нажать **«🎮 Запустить свой пак»** и выбрать файл пака `.sgpack`
   (или **«▶ Демо-пак»**, чтобы попробовать).
3. Приложение само поднимет сервер и войдёт ведущим.
   **Кнопку «Начать» пока не нажимать** — сначала пусть подключатся все игроки.

### 2. Узнать свой адрес в сети
Открыть терминал и выполнить:
```bash
hostname -I
```
Взять адрес вида **`192.168.x.x`** (или `10.x.x.x`) — обычно это первый адрес в
выводе. Этот адрес нужно сообщить игрокам.

> Пример: если команда вывела `192.168.1.42`, то игроки вводят `192.168.1.42`.

### 3. Начать партию
Когда все игроки появятся в списке — нажать **«Начать»**.

---

## Часть 2. Игроки (делает каждый у себя)

1. Открыть приложение → вкладка **«Игра»**.
2. Заполнить поля:
   - **адрес** → IP ведущего (например, `192.168.1.42`);
   - **порт** → `7777`;
   - **имя** → своё;
   - галочку **«я ведущий»** — **НЕ ставить**;
   - (по желанию) выбрать аватарку кнопкой **«🖼 Аватар»**.
3. Нажать **«Подключиться»**.
4. Дождаться, пока ведущий нажмёт «Начать». Готово!

---

## Если не получается подключиться

Идти по списку сверху вниз:

1. **Проверьте адрес.** Игроки вводят **IP ведущего**, а не `127.0.0.1`
   (localhost у каждого свой — так не сработает).

2. **Все в одной сети?** Один и тот же Wi-Fi/роутер у всех.

3. **Есть ли связь вообще.** Игрок может проверить в терминале:
   ```bash
   ping 192.168.x.x      # адрес ведущего
   ```
   Если не пингуется — проблема в сети, а не в игре (см. п. 5).

4. **Файрвол на компьютере ведущего.** Обычно на десктопном Linux выключен. Если
   включён `ufw`, ведущему открыть порты:
   ```bash
   sudo ufw allow 7777/tcp
   sudo ufw allow 7778/tcp
   ```

5. **«Изоляция клиентов» на Wi-Fi** — частая засада на **корпоративном или
   гостевом** Wi-Fi: устройства видят интернет, но **не видят друг друга**. Если
   адрес правильный и файрвол открыт, а соединения всё равно нет — почти наверняка
   это. Что делать: подключиться всем к **обычному домашнему роутеру** или к
   **точке доступа с телефона** (hotspot), либо соединиться по проводу.

---

## Шпаргалка (порты)

| Что | Порт |
|-----|------|
| Игра | `7777` |
| Медиа (картинки/видео/звук в вопросах) | `7778` |

Оба порта должны быть доступны на компьютере ведущего. Медиа игроки получают с
адреса ведущего автоматически — отдельно ничего настраивать не нужно.
