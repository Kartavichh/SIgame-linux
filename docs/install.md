# Установка и запуск SIGame-RS

Инструкция для Linux. Игра состоит из приложения-клиента (в нём играют) — этого
достаточно, чтобы **подключиться к игре**. Отдельный сервер нужен только
**ведущему** (см. последний раздел).

> Приложения собраны под **Linux**. Версий для Windows/macOS пока нет.
>
> **Требования:** Ubuntu **22.04+** (или аналог с glibc ≥ 2.35 и webkit2gtk-4.1).
> Если при запуске видите ошибку вида `GLIBC_2.39 not found` — значит сборку
> делали на слишком новой системе; пересоберите совместимо (см.
> [build-in-docker.sh](../scripts/build-in-docker.sh) — сборка на Ubuntu 22.04).

---

## Часть 1. Игрокам — установить клиент (это всё, что нужно)

### Где скачать
Открыть страницу релизов проекта:
**https://github.com/Kartavichh/SIgame-linux/releases** → последний релиз →
раздел **Assets**. Там два варианта клиента — выбрать **один**:

- `SIGame-RS_*.deb` — для **Ubuntu / Debian / Mint** (обычная установка).
- `SIGame-RS_*.AppImage` — для **любого** дистрибутива, без установки (один файл).

### Вариант A. `.deb` (Ubuntu/Debian) — рекомендуется
В терминале, в папке со скачанным файлом:
```bash
sudo apt install ./SIGame-RS_*.deb
```
После этого приложение **SIGame-RS** появится в меню программ. Запускать как
обычное приложение (или командой `sigame-client` в терминале).

Удалить потом, если надо:
```bash
sudo apt remove sigame-client
```

### Вариант B. AppImage (любой дистрибутив)
```bash
chmod +x SIGame-RS_*.AppImage     # сделать файл исполняемым (один раз)
./SIGame-RS_*.AppImage            # запустить
```
Никакой установки — просто файл, который можно положить куда угодно и запускать
двойным кликом или этой командой.

> Если AppImage не запускается с ошибкой про FUSE:
> `sudo apt install libfuse2` (Ubuntu 22.04+).

### Как подключиться к игре
Запустить приложение → вкладка **«Игра»** → ввести **адрес ведущего**, порт
`7777`, своё имя, **не** ставить галочку «я ведущий» → **«Подключиться»**.
Подробнее — в [lan-play.md](lan-play.md).

## Часть 2. Ведущему

Ведущий поднимает игру кнопкой **«🎮 Запустить свой пак»** в клиенте.

**Начиная с v0.3.3 сервер встроен в клиент** — отдельно ничего ставить не нужно,
кнопка работает сразу после установки `.deb`/AppImage.

> На более старых версиях (v0.3.2 и ранее) сервер в клиент не входил — там нужно
> было отдельно скачать `sigame-server` и положить в PATH. Просто обновитесь до
> последнего релиза.

### Запасной вариант: отдельный сервер в терминале
Если хочется держать сервер отдельным процессом (например, на выделенной машине),
можно скачать `sigame-server` из **Assets** релиза и запустить вручную, а в
клиенте подключиться к нему ведущим (адрес сервера, галочка «я ведущий»):
```bash
chmod +x sigame-server
./sigame-server путь/к/паку.sgpack
```
