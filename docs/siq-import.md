# Импорт паков `.siq` (родной формат SIGame) → `.sgpack`

Цель: дать возможность играть на тысячах готовых паков SIGame, конвертируя их в
наш формат `.sgpack`. Документ фиксирует разбор формата и решения по маппингу.

## Что такое `.siq`

`.siq` — это ZIP-архив (OPC-пакет, как `.docx`) со структурой:

```
content.xml          — манифест пакета (UTF-8 XML)
[Content_Types].xml  — служебный (OPC), при импорте игнорируем
Images/              — картинки
Audio/               — аудио
Video/               — видео
Texts/               — текстовые ресурсы (обычно метаданные, игнорируем)
```

**Важно: имена файлов в папках `Images/Audio/Video` URL-кодированы**
(пробел → `%20`, кириллица → `%D0%...`). При сопоставлении ссылок надо декодировать.

## Версии формата и объём v1

- **Классический** (`xmlns="http://vladimirkhil.com/ygpackage3.0.xsd"`,
  `version="3"/"4"`): вопрос описан через `<scenario>` из `<atom>`. **Поддерживаем.**
- **Новый** (`version="5"`, SIStorage): вопрос через `<params><param><item>`.
  **Поддерживается** (Этап 12d): `param name="question"` → слайды вопроса,
  `param name="answer"` → медиа ответа, `answerOptions` → варианты (подписанные
  слайды), медиа — `<item type="image|audio|video" isRef="True">имя`. Секретный
  вопрос (есть `param name="theme"`) → «кот в мешке».

Решение: детектим формат по наличию `<scenario>` (классика) vs `<params>` (новый);
оба пути строят одну и ту же модель и копируют медиа одинаково.

## Структура `content.xml` (классика)

```xml
<package name="..." version="4" date="..." publisher="...">
  <info><authors><author>Автор</author></authors></info>
  <rounds>
    <round name="Раунд 1" type="standart">
      <themes>
        <theme name="Тема">
          <questions>
            <question price="100">
              <scenario>
                <atom>Текст вопроса</atom>
                <atom type="image">@picture.jpg</atom>
                <atom type="voice">@sound.mp3</atom>
                <atom type="video">@clip.mp4</atom>
                <atom type="marker"/>          <!-- граница: дальше — ответ -->
                <atom type="image">@answer.jpg</atom>
              </scenario>
              <right><answer>Правильный ответ</answer></right>
              <wrong><answer>...</answer></wrong>   <!-- игнорируем -->
              <type name="auction"/>                <!-- особый тип, опц. -->
            </question>
          </questions>
        </theme>
      </themes>
    </round>
    <round name="Финал" type="final">…</round>
  </rounds>
</package>
```

## Маппинг на нашу модель (`core::pack`)

| `.siq`                                   | `.sgpack`                              |
|------------------------------------------|----------------------------------------|
| `package@name`                           | `Pack.name`                            |
| `info/authors/author` (через `, `)       | `Pack.author`                          |
| `round` (`type != "final"`)              | `Round` (обычный)                      |
| `round@type == "final"`                  | `Round.is_final = true`                |
| `theme@name`                             | `Theme.name`                           |
| `question@price`                         | `Question.price` (финал → 0)           |
| `scenario` (атомы до `marker`)           | `Question.question_slides`             |
| `right/answer` + атомы после `marker`    | `Question.answer_slides`               |
| `type@name`                              | `Question.kind` (см. ниже)             |

### Атомы → слайды

Сценарий SIGame проигрывается **последовательно**, поэтому **один `<atom>` = один
слайд** с одним блоком (это совпадает с нашей слайдовой моделью и листанием
ведущим). Типы атомов:

| `atom@type`        | блок `Content`        | папка медиа |
|--------------------|-----------------------|-------------|
| (нет) / `text`     | `Text`                | —           |
| `say`              | `Text` (реплика)      | —           |
| `image`            | `Image`               | `Images/`   |
| `voice`            | `Audio`               | `Audio/`    |
| `video`            | `Video`               | `Video/`    |
| `marker`           | граница вопрос/ответ  | —           |

- Содержимое медиа-атома вида `@file.ext` → ссылка на файл; берём из
  соответствующей папки (имя URL-декодируем), копируем байты в `media/` нашего
  пака под безопасным именем, в блоке пишем это имя.
- Медиа-атом без `@` (редко — встроенная ссылка/текст) обрабатываем как текст.

### Ответ

- `answer_slides` = слайд с текстом из `right/answer` (если несколько — соединяем
  через ` / `) **плюс** слайды из атомов после `marker` (медиа ответа).
- Если у вопроса нет ни текста ответа, ни атомов-ответа — `answer_slides` пустой
  (допустимо нашей моделью).

### Типы вопросов (`<type name=…>`)

| `.siq` `type@name`                  | `Question.kind` |
|-------------------------------------|-----------------|
| (нет) / `simple`                    | `Normal`        |
| `auction` / `stake`                 | `Auction`       |
| `cat` / `bagcat`                    | `CatInBag`      |
| `sponsored` / `noRisk` / `forAll`   | `NoRisk`        |

Параметры особых вопросов нового формата (стоимость кота, своя тема) в классике
обычно отсутствуют — берём только тип.

## Обработка ошибок и крайние случаи

- Не ZIP / нет `content.xml` → ошибка «не похоже на пак .siq».
- Новый формат (`<params>`) → ошибка с пояснением (см. выше).
- Ссылка на отсутствующий медиафайл → пропускаем атом, **накапливаем
  предупреждение** (импорт не падает); список предупреждений возвращаем в UI.
- Пустые темы/раунды переносим как есть (валидатор `.sgpack` затем подскажет).
- `<wrong>` (неверные варианты) игнорируем.

## API и интеграция

- `core`: модуль `siq.rs`, публичная функция
  `import_siq(path) -> Result<(PackArchive, Vec<String> /* предупреждения */)>`.
- `client`: Tauri-команда `import_siq(path)` → загружает в `EditorState`
  (как `editor_load`), возвращает пак + предупреждения; пользователь затем
  «Сохранить как… `.sgpack`» или сразу хостит. Кнопка «Импорт .siq» в редакторе.

## План реализации (подэтапы)

- **12a. `core::siq`** — парсер классического `.siq` → `PackArchive` + предупреждения;
  юнит-тесты на синтетическом `.siq` (текст, медиа, marker, типы, финал, новый
  формат → ошибка).
- **12b. Клиент** — команда `import_siq` + кнопка «Импорт .siq» в редакторе
  (загрузка → правка/сохранение/хостинг).
- **12c. Обкатка на реальных паках** — прогон на настоящих `.siq`, фиксы quirks.
