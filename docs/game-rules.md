# Правила игры

Документ описывает игровую механику. Обновляется по мере закрытия вопросов
раздела C из `TODO.md`.

## Краткая модель (контекст)

Игра идёт по **раундам**. В раунде — **табло**: темы (столбцы) ×
вопросы возрастающей стоимости (строки). Игрок выбирает клетку «тема + стоимость»,
открывается вопрос (текст/медиа), игроки жмут кнопку — кто первый, тот отвечает.
Верно → +стоимость; неверно → −стоимость, остальные снова могут жать.
В конце — финальный раунд с тайными ставками.

## C1. Объём правил v1 — РЕШЕНО

**Входит в v1:**
- Табло: темы × стоимости.
- Выбор клетки игроком/ведущим.
- Гонка кнопок (buzz-in): кто первый — тот отвечает.
- Начисление очков за верный ответ и штраф за неверный.
- Обычные вопросы с медиа (текст/изображение/видео).
- Финальный раунд с тайными ставками.

**Реализовано на Этапе 9 (было отложено на v2):**
- Особые типы вопросов: аукцион, «кот в мешке», вопрос без риска (см. ниже).

Причина: сначала играбельный «скелет» (выбор → вопрос → ответ → очки → финал),
затем наращиваем особые механики.

## Особые вопросы — РЕШЕНО (Этап 9)

Тип вопроса задаётся в паке полем `Question.kind`: `normal` (по умолчанию),
`auction`, `cat_in_bag`, `no_risk`. Часть правил настраивается ведущим в лобби
(структура `GameSettings`, команда `settings`) — по образцу настроек SIGame.

### Аукцион (`auction`)
Вопрос разыгрывается на ставки. Торгуются по очереди, начиная с выбравшего:
- Выбравший открывает торги ставкой **не ниже номинала** (пасовать на открытии
  нельзя). Дальше остальные по кругу: **повысить** ставку, **пас** или
  **ва-банк** (поставить весь свой счёт).
- Нельзя поставить больше своего счёта (кроме ва-банка = весь счёт). Кто не может
  перебить текущую ставку — автоматически выходит из торгов.
- Когда остаётся один участник, он отвечает **один**: верно `+ставка`, неверно
  `−ставка`. Гонки кнопок нет.
- Вырожденные случаи: один игрок или у выбравшего нет очков — он играет один за
  номинал.
- Команды: `bid {amount}`, `all_in`, `pass`.

### Кот в мешке (`cat_in_bag`)
Выбравший передаёт вопрос другому игроку (содержимое вопроса до передачи видит
только ведущий). Получивший отвечает **один**: верно `+номинал`, неверно
`−номинал`. Цена = номинал клетки.
- **Настройка `cat_must_give`** (по умолчанию вкл.): выбравший обязан отдать
  вопрос **другому** игроку. Если выключить — может оставить себе.
- Если отдать некому (нет других игроков) — играет сам.
- Команда: `give {target}`.

### Вопрос без риска (`no_risk`)
Отвечает только выбравший, гонки кнопок нет. Верно `+номинал`, неверно — **0**
(штрафа нет).
- **Настройка `no_risk_double`** (по умолчанию выкл.): при включении награда
  удвоенная (`+2×номинал`).

### Общее для особых вопросов
- Все они — «одиночный ответ» (`solo`): отвечает один игрок, при ошибке вопрос
  закрывается (не открывается снова). Верный ответ делает ответившего выбирающим;
  неверный — ход остаётся у первоначально выбравшего.
- Фазы движка: `Auction` (торги), `CatGive` (передача кота), затем `Answering`.

Реализация и тесты (`no_risk_*`, `cat_*`, `auction_*`): `core/src/game.rs`.
Сетевые команды и снимок — в `server/src/protocol.rs` (`AuctionView`,
`SettingsView`, поля `kind`/`solo`/`reward` в `CurrentView`).

## C2. Роли участников — РЕШЕНО

- **Ведущий (человек)** — он же хост. Ведёт игру, видит правильные ответы,
  управляет ходом. **Сам не играет.**
- **Игроки** — отвечают на вопросы.
- **Зрители** — в v1 не реализуем.

## C3. Проверка правильности ответа — РЕШЕНО

**Проверяет ведущий-человек** кнопками «верно / неверно».

Причина: надёжно, работает с любыми ответами (в т.ч. устными и по картинке).
Автоматическое сравнение текста отклонено как хрупкое
(«Пушкин» ≠ «А.С. Пушкин») и неприменимое к устным ответам.

## C4–C7 — РЕШЕНО (Этап 4, движок в `core::game`)

- **C4. Одновременные нажатия.** Первый успешно принятый «buzz» получает право
  ответа; на время ответа приём нажатий закрыт. Порядок одновременных нажатий
  определяет сервер (он принимает события по очереди). Ошибившийся блокируется
  на этом вопросе; для остальных кнопка открывается снова.
- **C5. Очки.** Верно `+стоимость`, неверно `−стоимость`. **Уход в минус разрешён.**
  Если ошиблись все игроки — вопрос закрывается, выбирающий не меняется.
  Угадавший становится следующим выбирающим.
- **C6. Таймеры.** Длительности (`buzz_time_secs`, `answer_time_secs`) хранятся в
  `GameConfig`, но движок время не отсчитывает. Сервер по истечении таймера
  вызывает `reveal()` (никто не нажал) или `answer_timeout()` (не ответил вовремя
  — засчитывается как неверный ответ).
- **C7. Автомат партии.** Фазы: `Lobby → Picking → Question → Answering →
  (Picking | RoundOver) → … → GameOver`. Команды: `add_player`, `start_game`,
  `pick`, `buzz`, `judge`, `reveal`, `answer_timeout`, `next_round`.
  Финальный раунд — отдельно (Этап 7).

Реализация и тесты: `core/src/game.rs`.

## Финальный раунд — РЕШЕНО (Этап 7): классический

Раунд помечается флагом `is_final` в паке (`Round.is_final`). Каждая тема
финального раунда — один вариант; играется первый вопрос оставшейся темы.
Стоимость вопросов в финале не используется.

Этапы финала (фазы движка):

1. **Вычёркивание тем** (`FinalThemeRemoval`). Участвуют игроки со счётом > 0.
   Они по очереди, **начиная с наименьшего счёта**, вычёркивают по одной теме,
   пока не останется одна. (Если тем ≤ 1 — этап пропускается.)
2. **Тайные ставки** (`FinalBets`). Каждый участник ставит от `1` до своего
   счёта. Ставки скрыты от других (виден только прогресс «сделали N из M»).
3. **Тайные ответы** (`FinalAnswers`). Каждый участник вводит ответ; ответы
   скрыты до вскрытия.
4. **Вскрытие** (`FinalReveal`). Ведущий вскрывает участников по одному
   (по возрастанию счёта), видит ответ и ставку, жмёт «верно/неверно»:
   верно — `+ставка`, неверно — `−ставка`. После последнего — `GameOver`.

Особенности:
- Игроки с **неположительным** счётом в финал не проходят (наблюдают).
- Если участников со счётом > 0 нет — финал пропускается, сразу `GameOver`.
- Правильный ответ финала виден только ведущему до самого вскрытия.

Команды движка: `remove_theme`, `final_bet`, `final_answer`, `final_judge`.
Реализация и тесты (`final_round_full_flow`, `final_skips_when_no_positive_players`):
`core/src/game.rs`. Сетевой снимок финала — в `server/src/protocol.rs` (`FinalView`).
