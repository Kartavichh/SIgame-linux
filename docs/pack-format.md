## D1. Свой формат паков — РЕШЕНО

Пак — **один файл — zip-архив** с расширением `.sgpack`. Внутри:

```
мой_пак.sgpack  (zip)
├── pack.json        # описание пака: темы, вопросы, ответы, ссылки на медиа
└── media/           # медиафайлы
    ├── img1.jpg
    ├── video1.webm
    └── ...
```

**Почему так:**
- Один файл удобно пересылать (это обычный архив).
- `pack.json` — текстовое «оглавление». JSON, т.к. и Rust (`serde`),
  и веб-фронтенд работают с ним без лишних усилий.
- Медиа лежат внутри архива в `media/`, `pack.json` ссылается на них по имени.

**Черновик структуры `pack.json`:**
```json
{
  "name": "Мой пакет",
  "author": "...",
  "rounds": [
    {
      "name": "Раунд 1",
      "themes": [
        {
          "name": "История",
          "questions": [
            {
              "price": 100,
              "content": [ { "type": "text", "value": "Кто..." } ],
              "answer": "..."
            }
          ]
        }
      ]
    }
  ]
}
```
**Схема `content` (реализовано в `core`, Этап 1):** массив элементов, каждый
с тегом `type`. Для медиа `value` — имя файла внутри `media/`:
```json
{ "type": "text",  "value": "Текст вопроса" }
{ "type": "image", "value": "tolstoy.jpg" }
{ "type": "video", "value": "clip.webm" }
{ "type": "audio", "value": "hint.mp3" }
```
Поле `format_version` в корне `pack.json` хранит версию формата (сейчас `1`)
для будущей совместимости. Поля `author`, `format_version`, `rounds` имеют
значения по умолчанию и в JSON необязательны.

## D2. Поддерживаемые медиа — РЕШЕНО

Форматы медиа определяет движок отображения. На Linux Tauri использует
WebKitGTK, у которого капризы с видео: MP4/H.264 может не воспроизводиться
без системных кодеков, а WebM работает стабильно.

| Тип      | Рекомендуемые форматы | Запасной (зависит от системы) |
|----------|-----------------------|-------------------------------|
| Картинки | PNG, JPEG, WebP, GIF  | —                             |
| Видео    | **WebM**              | MP4                           |
| Аудио    | OGG/Opus, MP3         | —                             |

- Авторам паков рекомендуем **WebM** для видео.
- MP4 тоже пытаемся отобразить, но без гарантий на всех системах.
- Жёсткого лимита размера пака пока нет; в редакторе позже можно
  предупреждать о слишком тяжёлых файлах.
