## Глобальный план реализации

Принцип: идём этапами, **каждый этап заканчивается чем-то видимым/запускаемым**.
Оставшиеся 🟡/🟢-вопросы решаем внутри соответствующего этапа (отмечено в скобках).
Особые механики, `.siq` и P2P вынесены в v2.

### Этап 0. Скелет проекта — ГОТОВО ✅
- [x] Создать Rust-workspace: крейты `core`, `server`, `client` (Tauri).
- [x] `cargo build` собирает все три крейта; `sigame-server` запускается и видит `core`.
- [x] Tauri-приложение запускается и открывает окно. _(финальная визуальная проверка `cargo tauri dev` — за пользователем)_
- **Готово, когда:** `cargo tauri dev` открывает окно.
- Фронтенд: чистый HTML/CSS/JS без сборщика (решено на старте этапа).

### Этап 1. `core`: модель паков
- [ ] Структуры данных пака (Pack/Round/Theme/Question/Content) + `serde`.
- [ ] Чтение и запись `.sgpack` (zip + `pack.json` + `media/`).
- [ ] Юнит-тесты на загрузку/сохранение.
- **Готово, когда:** пак читается из файла и пишется обратно без потерь.

### Этап 2. Просмотр пака в приложении (де-риск медиа) — ГОТОВО ✅
- [x] Tauri-команды: открыть `.sgpack`, отдать данные фронтенду.
- [x] Фронтенд показывает табло (темы × стоимости) и открывает вопрос.
- [x] Картинка, **видео (WebM)** и аудио воспроизводятся в окне.
- **Готово, когда:** реальный пак виден на экране, видео проигрывается. ✅
- **Важный вывод:** WebKitGTK НЕ играет медиа по схеме `asset://` (ошибка
  SRC_NOT_SUPPORTED). Медиа отдаём через встроенный HTTP-сервер на 127.0.0.1
  (`tiny_http`) с поддержкой Range. См. `docs/architecture.md`.

### Этап 3. Редактор паков — ГОТОВО ✅
- [x] UI: создание/редактирование раундов, тем и вопросов, ввод ответов.
- [x] Прикрепление медиа (нативный диалог) с превью, сохранение в `.sgpack`.
- [x] (🟡 D3) Объём редактора зафиксирован; валидация ссылок на медиа при сохранении
      (`validate_media`), очистка «осиротевших» медиа.
- **Готово, когда:** пак собирается с нуля в приложении и открывается в просмотре. ✅

### Этап 4. `core`: движок правил игры — ГОТОВО ✅
- [x] Конечный автомат партии: лобби → выбор клетки → вопрос → ответ → очки.
- [x] (🟡 C4) Разрешение одновременных нажатий (первый получает право ответа).
- [x] (🟡 C5) Подсчёт очков, уход в минус разрешён.
- [x] (🟡 C6) Таймеры в `GameConfig`; тайм-ауты через `reveal`/`answer_timeout`.
- [x] (🟢 C7) Полный автомат партии, 14 юнит-тестов проходят.
- **Готово, когда:** партию можно «проиграть» в тестах без UI и сети. ✅
- Реализация: `core/src/game.rs`. Решения C4–C7 — в `docs/game-rules.md`.

### Этап 5. Сетевой слой (headless-сервер + клиент) — ГОТОВО ✅
- [x] **Крейт `server` — headless-бинарь** `sigame-server`: запуск из терминала
      (`<пак.sgpack> [--port N]`), без GUI, владеет авторитетным `core::game::Game`.
- [x] (🟡 E1) Транспорт: **TCP + строки JSON** (без внешних крейтов). См. `docs/networking.md`.
- [x] (🟡 E2) Клиент подключается по `адрес:порт`; роль игрок/ведущий (`hello`).
- [x] (🟡 E3) Никнеймы, обрыв связи (счёт сохраняется), переподключение по имени.
- [x] Синхронизация: сервер шлёт полный снимок состояния при каждом изменении;
      правильный ответ виден только ведущему.
- [x] CLI-команды сервера: `status`, `players`, `start`, `help`, `quit`.
- **Готово, когда:** `sigame-server` запущен, к нему подключаются ведущий + игрок,
  состояние синхронизируется. ✅ Проверено сквозным тестом по TCP (старт → выбор →
  кнопка → вердикт; счёт, скрытие ответа, переподключение). Сборка обоих бинарей
  офлайн успешна. _GUI-проверка двумя окнами `cargo tauri dev` — за пользователем._

### Этап 6. Игровой режим (UI поверх сети) — ГОТОВО ✅
- [x] **Медиа по сети:** сервер раздаёт медиа по HTTP (`--media-port`, по умолчанию
      порт+1, с Range), клиент подставляет URL сервера в `<img>/<video>/<audio>`.
      Реализация: `server/src/media.rs`. Проверено (200/206/404, content-type).
- [x] Экран **ведущего**: табло-обзор, текст вопроса + **правильный ответ**,
      «кто отвечает», управление ходом (старт/верно/неверно/показать ответ/след. раунд).
- [x] Экран **игрока**: личный счёт, табло (выбор только в свой ход), большая
      кнопка ОТВЕТИТЬ, блокировка после ошибки (`locked_out`), статусы хода.
- [x] Полный игровой цикл обычных вопросов по сети.
- **Готово, когда:** можно сыграть реальную партию (без финала) с друзьями. ✅
  Проверено автотестом полной партии (2 игрока → game_over): выбор, гонка кнопок,
  блокировка, начисление/штраф, смена выбирающего, скрытие ответа от игроков.
  _Визуальная проверка экранов в GUI — за пользователем._

### Этап 7. Финальный раунд — ГОТОВО ✅
- [x] Классический финал: флаг `Round.is_final`; фазы вычёркивание тем →
      тайные ставки → тайные ответы → вскрытие ведущим с вердиктом.
- [x] Движок (`core/src/game.rs`) + 2 теста; протокол `FinalView`; UI экранов
      ведущего и игрока; редактор умеет помечать раунд финальным; демо-пак
      дополнен финальным раундом.
- **Готово, когда:** партия играется от начала до конца. ✅ Проверено сквозным
  сетевым тестом (обычный раунд → финал → game_over): вычёркивание по возрастанию
  счёта, скрытие ставок/ответов, начисление по ставкам. _GUI — за пользователем._
